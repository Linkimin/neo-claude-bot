Управляй **десктопным Claude Code** (Anthropic Agent SDK) с телефона через Telegram. Пишешь промпт в тему проекта — Claude Code выполняет его на твоём ПК, а потоковый ответ возвращается в чат: с инлайн-кнопками апрува, кнопкой прерывания, настройками на проект, продолжаемыми сессиями, обработкой лимитов и опциональным фолбэк-провайдером.

> Личный проект, ориентированный на Windows (служба «всегда онлайн» использует планировщик задач Windows). Ядро бота — обычный TypeScript и работает везде, где есть Node.

---

## Возможности

- **Тема на проект** — каждый проект это форум-топик в супергруппе; сообщения маршрутизируются в нужную рабочую директорию.
- **Живой стриминг** — текст ответа печатается в сообщение; вызовы инструментов показываются по мере выполнения.
- **Апрувы** — в режиме `default` опасные инструменты (Write/Edit/Bash) спрашивают разрешение инлайн-кнопками; если запрос висит — приходит напоминание.
- **Прерывание** — останови текущий промпт кнопкой ⏹ или `/stop` (настоящий `Query.interrupt()` по streaming-сессии, а не просто kill).
- **Настройки на проект** — режим разрешений, модель (Opus / Sonnet / Haiku), уровень «думанья» (effort), провайдер, фолбэк-модель.
- **Сессии** — диалог продолжается по проекту; `/new` начинает заново.
- **Лимиты** — определяет лимиты Claude, ставит запрос в очередь и авто-продолжает после сброса окна; проактивно пингует, когда лимиты вернулись.
- **Фолбэк модели** *(опционально)* — при упоре в лимиты Claude авто-переключение на запасной OpenAI-формат провайдер (через [claude-code-router](https://github.com/musistudio/claude-code-router)) и возврат после сброса.
- **Учёт трат** *(опционально)* — `/spend` показывает оценку расходов за день по проектам и баланс провайдера; алерты по дневным тратам и низкому балансу.
- **Всегда онлайн** — работает как служба Windows: стартует до входа в систему и авто-перезапускается при краше.

## Как это устроено

```
Telegram (телефон)
   │  промпт в теме проекта
   ▼
бот grammY  ──►  Core  ──►  runner  ──►  @anthropic-ai/claude-agent-sdk  ──►  Claude Code (ПК)
   ▲                │                          (streaming input + canUseTool)
   └── поток  ◄─────┘
       ответа / апрувы / статус
```

- **Бот** (`src/bot.ts`) — тонкий Telegram-слой: маршрутизация, клавиатуры, апрувы, команды.
- **Core** (`src/core.ts`) — оркестрирует прогон, владеет выбором провайдера и прерыванием.
- **runner** (`src/runner.ts`) — вызывает Agent SDK в режиме streaming-input, нормализует события, подключает колбэк апрува.
- **Сторы** — SQLite (`data/neo-claude-bot.sqlite`) для сессий, лимитов, прогонов и трат.
- **Фолбэк** — `CcrProcess` поднимает локальный прокси claude-code-router, который мостит OpenAI-формат провайдера в протокол Anthropic.

## Требования

- **Node.js ≥ 22**
- **Десктопный Claude Code**, единожды залогиненный (`claude` → `/login`). Бот наследует `~/.claude/.credentials.json`; API-ключа в проекте нет.
- **Telegram-бот** (от [@BotFather](https://t.me/BotFather)) и **форум-супергруппа** (группа с включёнными *Темами*), где ты админ.
- Windows нужен только для службы «всегда онлайн»; разработка идёт на любой ОС.

## Установка

```bash
npm install
cp .env.example .env      # заполнить переменные ниже
npm start                 # запуск на переднем плане
```

В Telegram:

1. Добавь бота в форум-супергруппу администратором (разреши управление темами).
2. Отправь `/setup` — он создаст по теме на каждый проект из `config/projects.json`.
3. Открой тему проекта и пиши промпт.

### Переменные окружения (`.env`)

| Переменная | Обязательна | Назначение |
|---|---|---|
| `TELEGRAM_BOT_TOKEN` | ✅ | Токен от BotFather. |
| `TELEGRAM_USER_ID` | ✅ | Твой числовой Telegram-id — единственный разрешённый пользователь. |
| `TELEGRAM_GROUP_ID` | ✅ | Id форум-супергруппы (отрицательный, напр. `-1001234567890`). |
| `SETTINGS_PIN` | ✅ | PIN для разблокировки auto/bypass-режима (`/auto <PIN>`). |
| `ROUTERAI_API_KEY` | — | Включает фолбэк на запасного провайдера. |
| `ROUTERAI_BASE_URL` | — | Базовый URL провайдера (OpenAI-формат). |
| `CCR_PORT` | — | Порт локального claude-code-router (по умолчанию `3456`). |
| `SPEND_ALERT_USD` | — | Порог алерта по дневным тратам. |
| `ROUTERAI_BALANCE_MIN` | — | Порог алерта по низкому балансу. |

### Проекты (`config/projects.json`)

```json
[
  { "name": "myapp", "dir": "D:/work/myapp", "defaultMode": "acceptEdits" }
]
```

`defaultMode` — один из `default` (спрашивать перед опасными инструментами), `acceptEdits` (авто-принимать правки) или `bypassPermissions` (выполнять всё — разблокировать с бота через `/auto <PIN>`).

## Команды

| Команда | Что делает |
|---|---|
| `/setup` | Создать форум-тему на каждый проект. |
| `/status` | Проекты, текущая модель/режим/effort, идёт ли прогон. |
| `/settings` | Инлайн-меню: режим, модель, effort, провайдер, фолбэк. |
| `/new` | Начать новую сессию в этой теме. |
| `/stop` | Прервать текущий промпт (как кнопка ⏹). |
| `/limits` | Остаток лимитов Claude. |
| `/spend` | Оценка трат за день + баланс провайдера. |
| `/auto <PIN>` | Включить bypass-режим для этого проекта. |

## Запуск как службы Windows (всегда онлайн)

Из PowerShell **от администратора**, в каталоге проекта:

```powershell
npm run install-service                  # установить (старт при загрузке, авто-рестарт)
Start-ScheduledTask -TaskName NeoClaudeBot   # запустить сейчас (и при загрузке)
npm run uninstall-service                # удалить
```

- Работает **до входа в систему** под твоим аккаунтом (S4U) — Claude-авторизация работает.
- Лог: `data\bot.log`. Статус: `Get-ScheduledTask -TaskName NeoClaudeBot`.
- Супервизор-обёртка перезапускает бота, если он завершился.

## Разработка

```bash
npm test            # vitest
npx tsc --noEmit    # проверка типов
```

Исходники — небольшие сфокусированные модули в `src/` с соседними `*.test.ts`. Чистая логика (рендер, нормализация, маршрутизация, сторы) покрыта юнит-тестами; живые пути Telegram/SDK проверяются вручную.

## Модель безопасности

- **Allowlist на одного** — взаимодействовать может только `TELEGRAM_USER_ID`, остальные отклоняются.
- **Bypass под PIN** — `bypassPermissions` (выполнять любой инструмент без вопросов) требует `/auto <PIN>`.
- **Секретов в репо нет** — учётные данные в `.env` (git-ignore) и в `~/.claude`.

Относись к боту как к удалённому shell в свою машину: запускай только в чате, который контролируешь, и держи `bypassPermissions` для проектов, которым доверяешь.
